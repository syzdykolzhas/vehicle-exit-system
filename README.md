# Система управления выходом автотранспорта

Современная веб-система для учета и управления автотранспортом предприятия с аналитикой по подразделениям.

## Функциональность

### Основные возможности

- **Управление автопарком**: 160 единиц транспорта
- **Учет по подразделениям**: Каждое подразделение имеет от 3 до 19 транспортных средств
- **Система заявок**: Создание и отслеживание заявок на выход транспорта
- **Аналитика и статистика**: Детальная аналитика по дням и месяцам
- **Фильтрация**: По подразделениям и датам

### KPI показатели на дашборде

- **Всего транспорта**: Общее количество транспортных средств
- **На ремонте**: Количество ТС на ремонте
- **С заявками**: Количество ТС с поданными заявками
- **Без заявок**: Количество ТС без заявок
- **Отклонено**: Количество отклоненных заявок
- **Выход сегодня**: Процент выхода транспорта на текущий день

### Страницы системы

1. **Дашборд** - Основная страница с KPI картами и графиками
2. **Подразделения** - Управление структурными подразделениями
3. **Транспорт** - Список и управление транспортными средствами
4. **Заявки** - Создание и управление заявками на выход
5. **Аналитика** - Детальная аналитика с графиками

### Аналитика

- **Процент выхода по дням**: График динамики выхода за выбранный период
- **Распределение по статусам**: Круговая диаграмма статусов заявок
- **Заявки по подразделениям**: Распределение заявок между подразделениями
- **Статистика по подразделениям**: Таблица с детальной статистикой по каждому подразделению

## Установка и запуск

Система не требует установки зависимостей. Для работы нужен только современный веб-браузер.

1. Откройте файл `index.html` в браузере
2. Система автоматически загрузит демо-данные при первом запуске

## Технологии

- **HTML5** - Структура приложения
- **CSS3** - Стилизация с современным дизайном
- **JavaScript (Vanilla)** - Логика приложения
- **Chart.js** - Графики и диаграммы
- **LocalStorage** - Хранение данных в браузере

## Структура данных

### Подразделения (Departments)
- ID
- Название
- Количество транспорта (3-19)

### Транспорт (Vehicles)
- ID
- Гос. номер
- Марка/Модель
- ID подразделения
- Статус ремонта

### Заявки (Requests)
- ID
- Дата
- ID транспортного средства
- ID подразделения
- Статус (approved/rejected/pending)
- Цель поездки

## Хранение данных

Все данные сохраняются в LocalStorage браузера. При первом запуске система автоматически генерирует демо-данные:
- 12 подразделений
- 160 транспортных средств
- Заявки за последние 30 дней

## Адаптивность

Система полностью адаптивна и работает на:
- Десктопах
- Планшетах
- Мобильных устройствах
